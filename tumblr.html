Continuous Integration

Local → GitHub → Testes → Aplicação

Utilizando Continuous Integration a cada vez que o desenvolvedor tiver algum valor a acrescer no projeto ele o integrará com o código principal. Esse repositório que por sua vez fica "vigiado" por outra ferramenta, que irá executar os testes no código implementados.
Por fim, se tudo estiver nos conformes, a pequena parte de código enviada pelo desenvolvedor se integrará a principal.

Elevator Pitch

Boa impressão em menos de um minutos


Apresente seu produto no tempo em que o elevador vai do térreo ao oitavo andar.

30 segundo é o suficiente quando o contexto é objetivo.

Elevator Pitch é particionado para que seja simples de ser construído e fácil de ser apresentado:

  • Paro o(a): [publico alvo];
  • Que tem: [necessidade do público-alvo];
  • O(a): [nome do produto];
  • É um(a): [categoria do mercado];
  • Que: [benefício principal];
  • Ao contrário do(a): [concorrente(s)];
  • O produto: [diferencial do produto];

Este não é o único modo, mas no meu ponto de vísta é o mais efetivo.
